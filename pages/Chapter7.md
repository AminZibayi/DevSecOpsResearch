# Agent Identity and Access Management (IAM)

To maintain a zero-trust architecture, agents must have distinct, auditable identities.

<div class="grid grid-cols-2 gap-5 mt-6">
  <div>
    <h3 class="font-bold text-[var(--slidev-theme-accents-teal)] text-xl mb-3 border-b border-gray-600 pb-2">Unique Agent IDs</h3>
    <p class="text-sm text-gray-300">Agents (e.g., Claude Code, CI bots) must operate under Service Accounts or GitHub Apps rather than sharing human credentials, satisfying <strong>HIPAA §164.312(a)(2)(i)</strong> and <strong>PCI DSS 8.2.1</strong>.</p>
  </div>
  <div>
    <h3 class="font-bold text-[var(--slidev-theme-accents-teal)] text-xl mb-3 border-b border-gray-600 pb-2">Short-Lived Tokens</h3>
    <p class="text-sm text-gray-300">Temporary tokens with minimum permissions (e.g., `contents:read`) ensure agents only access what they explicitly require for the task.</p>
  </div>
  <div>
    <h3 class="font-bold text-[var(--slidev-theme-accents-teal)] text-xl mb-3 border-b border-gray-600 pb-2">Automatic Logoff</h3>
    <p class="text-sm text-gray-300">Harnesses must be configured to terminate active sessions after inactivity, complying with HIPAA session mandates.</p>
  </div>
  <div>
    <h3 class="font-bold text-[var(--slidev-theme-accents-teal)] text-xl mb-3 border-b border-gray-600 pb-2">Human-in-the-Loop (HITL)</h3>
    <p class="text-sm text-gray-300">Mandatory human approval is required for high-risk actions like privilege elevation or infrastructure deletion.</p>
  </div>
</div>

# Audit Trails for AI-Originated Changes

Accountability requires tracing production changes back to the original intent and reasoning, directly satisfying GDPR Recital 39 mandates.

<div class="mt-3 space-y-3">
  <div class="bg-[var(--slidev-theme-code-background)] p-3 rounded-lg border-l-4 border-[var(--slidev-theme-accents-yellow)]">
    <h3 class="font-bold text-xl text-white mb-2">The "Intent-to-Artifact" Chain</h3>
    <ul class="text-sm space-y-2 list-disc ml-6 text-gray-300">
      <li><strong>Prompt Logging:</strong> Capture and store prompts in an immutable log (`aidefence_stats`) to allow auditors to understand the agent's proposal context.</li>
      <li><strong>Reasoning Metadata:</strong> Attach the agent's step-by-step reasoning (Chain of Thought) directly to the Pull Request.</li>
      <li><strong>Commit Labeling:</strong> Use trailers like `Co-authored-by: Claude &lt;claude@anthropic.com&gt;` and labels (`ai-generated`) to distinguish changes.</li>
    </ul>
  </div>

  <div class="bg-[var(--slidev-theme-code-background)] p-3 rounded-lg border-l-4 border-[var(--slidev-theme-accents-blue)]">
    <h3 class="font-bold text-xl text-white mb-2">Non-Repudiation via Signed Commits</h3>
    <p class="text-sm text-gray-300">Mandating cryptographic signing (Sigstore/Cosign, GPG) proves a commit was generated by a verified pipeline run and hasn't been tampered with post-generation.</p>
  </div>
</div>

AI-native workflows accelerate compliance via automated evidence collection.

    <ul class="text-sm space-y-4 text-gray-300">

# Global Compliance Mapping: GDPR

<div class="mt-3 bg-[var(--slidev-theme-code-background)] p-4 rounded-lg border-t-4 border-[var(--slidev-theme-accents-teal)]">
  <h3 class="font-bold text-2xl text-white mb-4">GDPR (Data Protection by Design & Default)</h3>
  <ul class="text-sm space-y-3 text-gray-300">
    <li><strong>Transparency & Rights (Art 17/20):</strong> Agents implement Data Portability and Erasure endpoints as part of initial feature generation.</li>
    <li><strong>Data Minimization:</strong> Agents constrained to query specific columns rather than using `SELECT *`, satisfying data minimization principles.</li>
    <li><strong>Security of Processing (Art 32):</strong> Pseudonymization and encryption are implemented as default coding constraints in the system prompt.</li>
    <li><strong>The 3-Gate Pattern:</strong> Runtime PII detection (Gate 1) prevents sensitive data from leaking into logs or LLM training sets.</li>
  </ul>
